## Команды
---
### 1. **Клонирование репозитория**

Скачивает удалённый репозиторий на локальный компьютер.

```bash
git clone <URL>
```

---

### 2. **Инициализация нового репозитория**

Создаёт новый локальный репозиторий.

```bash
git init
```

---

### 3. **Добавление изменений и отправка в удалённый репозиторий**

Включает процесс добавления файлов, фиксации изменений и отправки в удалённый репозиторий.

```bash
git add .
git commit -m "Ваш комментарий"
git push origin <ветка>
```

---

### 4. **Обновление локального репозитория**

Получает последние изменения из удалённого репозитория.

```bash
git pull origin <ветка>
```

---

### 5. **Сравнение изменений**

Сравнивает изменения в файлах перед их фиксацией.

```bash
git diff
```

Для сравнения с предыдущим коммитом:

```bash
git diff HEAD
```

---

### 6. **Просмотр статуса репозитория**

Показывает, какие файлы изменены или ещё не добавлены.

```bash
git status
```

---

### 7. **Просмотр истории коммитов**

Отображает список предыдущих коммитов.

```bash
git log
```

Сжатый список коммитов:

```bash
git log --oneline
```

---

### 8. **Создание новой ветки**

Создаёт и переключается на новую ветку.

```bash
git checkout -b <имя_ветки>
```

Отправка ветки в удалённый репозиторий:

```bash
git push origin <имя_ветки>
```

---

### 9. **Переключение между ветками**

Переходит на указанную ветку.

```bash
git checkout <имя_ветки>
```

---

### 10. **Слияние веток**

Объединяет указанную ветку с текущей.

```bash
git merge <имя_ветки>
```

---

### 11. **Удаление ветки**

Локально:

```bash
git branch -d <имя_ветки>
```

Удалённо:

```bash
git push origin --delete <имя_ветки>
```

---

### 12. **Удаление файла из Git (не из файловой системы)**

```bash
git rm --cached <имя_файла>
git commit -m "Удалён файл из индекса"
git push
```

---

### 13. **Отмена изменений**

Отменяет изменения в файле, не зафиксированные коммитом.

```bash
git checkout -- <имя_файла>
```

Отмена последнего коммита (изменения остаются):

```bash
git reset HEAD~
```

---

### 14. **Создание тегов**

Добавляет тег к указанному коммиту (например, для версии).

```bash
git tag -a <имя_тега> -m "Комментарий"
git push origin <имя_тега>
```

---

### 15. **Работа с подмодулями**

Добавление подмодуля:

```bash
git submodule add <URL>
```

Обновление подмодулей после клонирования:

```bash
git submodule update --init --recursive
```

---

### 16. **Чистка репозитория**

Удаление ненужных файлов и кэшированных данных:

```bash
git clean -f
```

Удаление локальных веток, не синхронизированных с удалённым репозиторием:

```bash
git fetch --prune
```

---

### 17. **Восстановление удалённого коммита**

Если коммит был случайно удалён:

```bash
git reflog
git checkout <хэш_коммита>
```

---

### 18. **Сжатие истории коммитов**

Объединение нескольких коммитов в один:

```bash
git rebase -i HEAD~<число_коммитов>
```

---

### 19. **Просмотр файла из прошлого коммита**

Посмотреть состояние файла из определённого коммита:

```bash
git show <хэш_коммита>:<путь_к_файлу>
```

---

### 20. **Решение конфликтов слияния**

1. Выполните слияние, чтобы вызвать конфликт:
```bash
git merge <ветка>
```
2. Исправьте конфликт вручную в файлах.
3. Добавьте изменения:
```bash
git add <имя_файла>
```
4. Завершите слияние:
```bash
git commit
```
